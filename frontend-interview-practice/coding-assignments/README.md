# Frontend Interview Practice

A collection of real-world frontend implementation challenges organized by difficulty and type.

## How to Use

1. Each challenge has a time limit (noted in the header)
2. Try to complete without looking at solutions first
3. Focus on: functionality → clean code → polish
4. Test your work manually before considering it complete

## Challenge Categories

### Basic (30-45 min each)
- `01-todo-list/` - Classic todo with CRUD operations
- `02-counter-with-history/` - Counter with undo/redo
- `03-tabs-component/` - Tab navigation UI
- `04-accordion/` - Collapsible accordion panels
- `05-modal-dialog/` - Accessible modal component

### Intermediate (45-60 min each)
- `06-autocomplete/` - Search with debounced API calls
- `07-infinite-scroll/` - Paginated list with infinite scroll
- `08-image-carousel/` - Image slider with controls
- `09-form-validation/` - Multi-field form with validation
- `10-drag-and-drop/` - Draggable list items

### Advanced (60-90 min each)
- `11-data-table/` - Sortable, filterable data table
- `12-kanban-board/` - Trello-like board with drag-drop
- `13-markdown-editor/` - Live preview editor
- `14-chat-interface/` - Real-time messaging UI
- `15-dashboard/` - Multi-widget analytics dashboard

## Tech Stack: React + Vite

All challenges use React with Vite for a modern development experience.
This matches most real-world interview environments.

## Skills Tested

- ✅ DOM manipulation
- ✅ Event handling
- ✅ State management
- ✅ API integration
- ✅ Async operations
- ✅ CSS layouts
- ✅ Responsive design
- ✅ Accessibility
- ✅ Performance optimization

## Tips

- Read requirements fully before coding
- Plan your approach (2-3 min)
- Start with core functionality
- Add error handling
- Test edge cases
- Consider accessibility last

## Quick Start

```bash
# Pick any challenge
cd 01-todo-list

# Install dependencies
npm install

# Start development server
npm run dev

# Open http://localhost:5173 and start coding!
```
