## üí° Key Patterns & Learnings

Today's key insights:

1. **Sliding Window - Variable Size**: Minimum Size Subarray Sum uses window that expands until sum >= target, then contracts from left until sum < target. Track minimum length when condition is met. O(n) time because each element added/removed at most once.

2. **Sliding Window - Fixed Size**: Permutation in String uses fixed-size window (length of s1). Slide window across s2, compare character counts. Optimization: Track matches count instead of comparing entire arrays.

3. **React State - useReducer Pattern**: useReducer simplifies complex state logic, especially when:
   - Next state depends on previous state
   - Multiple state pieces update together
   - Complex validation logic (forms)
   - Benefits: Predictable state updates, easier to test, cleaner action logs

4. **Component Patterns - Tabs vs Accordion**: Both use index-based state but differ in complexity:
   - Tabs: Simple, no animation needed, ARIA attributes straightforward
   - Accordion: Requires height animation (0 ‚Üí auto tricky), more content handling
   - Shared pattern: Compound component structure (Parent + children)

5. **Accessibility Planning**: ARIA attributes should be designed from start, not added later. Tabs need role="tablist"/"tab"/"tabpanel", aria-selected, aria-controls. Keyboard navigation (arrow keys) important for usability.

6. **Integrated Learning Effectiveness**: Learn ‚Üí Apply ‚Üí Explain structure reinforced understanding:
   - Morning: Read docs on state management
   - Afternoon: Built form with useReducer, struggled through implementation
   - Evening: Explained reasoning aloud, identified gaps in Context knowledge
