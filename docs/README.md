## üìù Contributing to Documentation

### Adding New Documents

1. **Create in appropriate folder**:
   - General guides ‚Üí `docs/`
   - Schedule-related ‚Üí `schedules/`
   - Daily tracking ‚Üí `daily-logs/`
   - Company-specific ‚Üí `company-prep/`
   - Challenge-related ‚Üí `frontend-interview-practice/`

2. **Follow naming convention**:
   - Use descriptive names: `Topic_Name_Guide.md`
   - Use underscores for spaces
   - Use PascalCase for titles

3. **Include standard sections**:
   ```markdown
   # Document Title

   Brief description

   ---

   ## Section 1
   Content...

   ## Section 2
   Content...

   ---

   ## Related Resources
   Links to related docs
   ```

4. **Update relevant README**:
   - Add to folder's README.md
   - Link from main README if important
